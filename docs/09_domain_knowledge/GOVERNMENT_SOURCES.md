## HOW TO UPDATE WHEN IRCC CHANGES SOMETHING

1. Check the relevant government source URL above
2. Compare with current `config/programs.yaml` or `config/questionnaires.yaml`
3. Update the YAML file with the new information
4. Add a `last_verified` comment in the YAML file
5. Update this document with the new verification date
6. Push to GitHub → Railway auto-deploys in 90 seconds
7. No code changes needed — config-driven architecture handles it

**Architecture ensures easy maintenance**: All IRCC-dependent data lives in YAML config files, not in Python code. When IRCC changes a form or processing time, edit the YAML and push. No developer needed.
